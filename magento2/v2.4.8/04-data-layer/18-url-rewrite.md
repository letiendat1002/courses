# URL Rewrite & Store-Based Routing

Every URL in Magento goes through a rewrite resolution process. The `url_rewrite` table is the map that tells Magento how to convert a human-readable URL into an internal path. Understanding this system is essential for SEO, multi-store setups, and any feature that generates dynamic URLs.

---

## 1. The URL Rewrite Table

### Table Structure

```sql
CREATE TABLE url_rewrite (
    url_rewrite_id       INT PRIMARY KEY AUTO_INCREMENT,
    entity_type         VARCHAR(50) NOT NULL,         -- 'category', 'product', 'cms-page', 'custom'
    entity_id           INT NOT NULL,
    request_path        VARCHAR(255) NOT NULL,        -- The URL seen by user: 'women/jackets.html'
    target_path         VARCHAR(255) NOT NULL,       -- Internal path: 'catalog/category/view/id/12'
    redirect_code       INT DEFAULT 0,                -- 0=direct, 301=permanent, 302=temporary
    store_id            INT NOT NULL,
    description          VARCHAR(255) NULL,
    is_autogenerated     TINYINT(1) DEFAULT 0,
    metadata            JSON NULL
);
```

### Index Strategy

```sql
-- Critical indexes for URL rewrite performance
CREATE INDEX IDX_URL_REWRITE_STORE_ID_REQUEST_PATH ON url_rewrite(store_id, request_path);
CREATE INDEX IDX_URL_REWRITE_ENTITY_TYPE_ENTITY_ID ON url_rewrite(entity_type, entity_id);
```

### How a URL Gets Rewritten

```
User visits: https://example.com/women/jackets.html
                 ↓
Router checks url_rewrite table:
  WHERE store_id = 1 AND request_path = 'women/jackets.html'
                 ↓
Found: target_path = 'catalog/category/view/id/12'
                 ↓
Magento dispatches to: Catalog\Controller\Category\View::execute()
with $categoryId = 12
```

## 2. URL Resolution Process

### The Full Resolution Chain

```php
<?php
// \Magento\Framework\Url::getUrl()
// Called whenever Magento generates any URL

public function getUrl($routePath, $routeParams = [])
{
    // Step 1: Determine store ID from context
    $storeId = $this->_getStoreId();

    // Step 2: Process route to internal path
    $routePath = $this->routeFactory->getRoute($routePath, $routeParams);

    // Step 3: Check url_rewrite for this store
    $rewrite = $this->urlRewriteFinder->findByRequestPath(
        $routePath,
        $storeId
    );

    if ($rewrite) {
        // Step 4: If rewrite found, use target_path
        return $this->processRewrite($rewrite);
    }

    // Step 5: No rewrite → use standard routing
    return $this->standardRoute($routePath, $routeParams);
}
```

### Request Path Resolution Flow

```php
<?php
// \Magento\UrlRewrite\Model\UrlFinder
public function findByRequestPath($requestPath, $storeId)
{
    $connection = $this->getConnection();

    $select = $connection->select()->from(['ur' => $this->getTable('url_rewrite')])
        ->where('request_path = ?', $requestPath)
        ->where('store_id = ?', $storeId);

    return $connection->fetchRow($select);
}
```

### Store-Specific Resolution

The `store_id` column is critical — a URL rewrite for store 1 does NOT apply to store 2:

```sql
-- Store 1 (English) has different URL than Store 2 (German)
INSERT INTO url_rewrite (entity_type, entity_id, request_path, target_path, store_id)
VALUES ('product', 42, 'en/women/jackets.html', 'catalog/product/view/id/42', 1);

INSERT INTO url_rewrite (entity_type, entity_id, request_path, target_path, store_id)
VALUES ('product', 42, 'de/damen/jacken.html', 'catalog/product/view/id/42', 2);
```

When a customer is in the German store view, only `de/damen/jacken.html` resolves — the English URL returns 404 for that store.

## 3. Generating URLs in Code

### Core URL Generation Methods

```php
<?php
// Inject \Magento\Framework\UrlInterface or use UrlFactory

// Basic product URL (respects URL rewrites)
$productUrl = $this->url->getUrl('catalog/product/view', [
    'id' => $product->getId()
]);

// Category URL
$categoryUrl = $this->url->getUrl('catalog/category/view', [
    'id' => $category->getId()
]);

// CMS Page URL
$cmsUrl = $this->url->getUrl('cms/page/view', [
    'page_id' => $pageId
]);

// Direct route URL
$directUrl = $this->url->getUrl('training/review/view', [
    'review_id' => $reviewId
]);
```

### URL with Store Context

```php
<?php
// Generate URL for a specific store (useful in multi-store contexts)
$ germanStoreUrl = $this->url->getUrl('catalog/product/view', [
    'id' => $product->getId(),
    '_scope' => $germanStoreId  // Forces store context
]);

// Or with store object directly
$ germanStoreUrl = $this->url->getUrl('catalog/product/view', [
    'id' => $product->getId(),
    '_scope' => $germanStore    // \Magento\Store\Api\Data\StoreInterface
]);

// For current store context (default)
$currentStoreUrl = $this->url->getUrl('catalog/product/view', [
    'id' => $product->getId(),
    '_scope' => true  // Use current store
]);
```

### URL with Additional Parameters

```php
<?php
// Query string parameters
$url = $this->url->getUrl('catalog/product/view', [
    'id' => $product->getId(),
    'utm_source' => 'email',
    'utm_medium' => 'newsletter'
]);
// Result: /catalog/product/view/id/42?utm_source=email&utm_medium=newsletter

// Fragment (anchor)
$url = $this->url->getUrl('catalog/product/view', [
    'id' => $product->getId(),
    '_fragment' => 'reviews'
]);
// Result: /catalog/product/view/id/42#reviews

// Force HTTPS
$secureUrl = $this->url->getUrl('customer/account/login', [
    '_secure' => true
]);
```

## 4. Canonical URLs and SEO

### How Canonical URLs Work

```php
<?php
// Get canonical URL for a product (without session/parameters)
$canonical = $product->getUrlKey();  // From url_rewrite request_path

// In templates, use:
<link rel="canonical" href="<?= $block->escapeUrl($product->getProductUrl()) ?>" />
```

### Canonical URL Best Practices

- Always generate canonical URLs from the `url_rewrite` table, not hardcoded paths
- In multi-store setups, canonical URLs should point to the correct store domain
- Never include `___store` parameter in canonical URLs (it bypasses normal routing)

### Store-Specific Canonical Generation

```php
<?php
// Generate canonical URL for a specific store
public function getCanonicalUrl(\Magento\Catalog\Api\Data\ProductInterface $product, int $storeId): string
{
    // Get URL rewrite for this store
    $rewrite = $this->urlFinder->findByEntityIdAndStore(
        'product',
        $product->getId(),
        $storeId
    );

    if ($rewrite) {
        $store = $this->storeManager->getStore($storeId);
        return $store->getBaseUrl() . $rewrite->getRequestPath();
    }

    // Fallback to URL key
    return $this->url->getUrl('catalog/product/view', [
        'id' => $product->getId(),
        '_scope' => $storeId
    ]);
}
```

## 5. Custom URL Rewrites

### Creating Programmatic Rewrites

```php
<?php
// Use \Magento\UrlRewrite\Model\UrlRewriteFactory
public function __construct(
    \Magento\UrlRewrite\Model\UrlRewriteFactory $urlRewriteFactory,
    \Magento\Store\Model\StoreManagerInterface $storeManager
) {
    $this->urlRewriteFactory = $urlRewriteFactory;
    $this->storeManager = $storeManager;
}

public function createCustomRewrite(int $productId, string $requestPath, int $storeId): void
{
    /** @var \Magento\UrlRewrite\Model\UrlRewrite $urlRewrite */
    $urlRewrite = $this->urlRewriteFactory->create();

    $urlRewrite->setEntityType('custom')
        ->setEntityId($productId)
        ->setRequestPath($requestPath)
        ->setTargetPath('catalog/product/view/id/' . $productId)
        ->setRedirectCode(0)  // 0 = permanent redirect, not direct
        ->setStoreId($storeId)
        ->setDescription('Custom rewrite for product landing page')
        ->save();
}
```

### Custom Rewrite for Landing Pages

```php
<?php
// Create a redirect from old URL to new product
$urlRewrite = $this->urlRewriteFactory->create();
$urlRewrite->setEntityType('custom')
    ->setEntityId(0)  // No specific entity
    ->setRequestPath('summer-sale')
    ->setTargetPath('catalog/search/result?q=summer+deal')
    ->setRedirectCode(302)  // Temporary redirect
    ->setStoreId($storeId)
    ->save();
```

### Deleting Rewrites

```php
<?php
// Delete specific URL rewrite
public function deleteRewrite(string $requestPath, int $storeId): void
{
    $urlRewrite = $this->urlRewriteFactory->create();
    $urlRewrite->loadByRequestPath($requestPath, $storeId);
    if ($urlRewrite->getId()) {
        $urlRewrite->delete();
    }
}
```

## 6. URL Rewrites for Products

### Product URL Key Generation

Product URL keys are auto-generated from the product name when `use_categories` setting is configured:

```sql
-- Key generation based on configuration:
-- catalog/seo/product_url_suffix = '.html'
-- catalog/seo/use_categories = '1'

-- URL rewrite for product in category:
-- /women/apparel/jackets.html
-- instead of /jackets.html
```

### Product URL in Different Store Views

```php
<?php
// Each store can have different URL key for the same product
// Get URL for specific store's URL key:
$product->setStoreId($storeId);
$urlKey = $product->getUrlKey();

// Or via repository:
$urlRewrite = $this->urlFinder->findByEntityIdAndStore(
    'product',
    $productId,
    $storeId
);
$requestPath = $urlRewrite->getRequestPath();
```

### Product URL Invalidation

When a product or category changes, URL rewrites are automatically regenerated by the `url_rewrite_indexer`. This runs on:
- Product save (if URL key changed)
- Category save (if URL key or path changed)
- When running `bin/magento indexer:reindex url_rewrite`

## 7. URL Rewrites for Categories

### Category URL Path

Categories maintain a hierarchical URL path:

```sql
-- Category tree:
-- Root (id=1)
--   ├── Men (id=2, url_key='men', path='men')
--   │   ├── Apparel (id=5, url_key='apparel', path='men/apparel')
--   │   │   ├── Jackets (id=12, url_key='jackets', path='men/apparel/jackets')

-- URL rewrite for category 12:
-- request_path = 'men/apparel/jackets.html'
-- target_path = 'catalog/category/view/id/12'
```

### Anchoring Categories to Root

Categories can be assigned a "root" that limits their URL to a specific parent:

```php
<?php
// Set category to use anchored URL (from its parent in tree)
$category->getResource()->setUrlPath($category, 'men/apparel/jackets');
```

## 8. URL Rewrite Generation Process

### Automatic Generation (Indexer)

```php
<?php
// \Magento\UrlRewrite\Model\Storage\DbStorage
// The url_rewrite indexer generates rewrites for:
// 1. Products (when saved or reindexed)
foreach ($products as $product) {
    $this->saveUrlRewrite($product->toArray(), $product->getUrlKey());
}

// 2. Categories (when saved or reindexed)
// 3. CMS pages (when saved)
```

### Manual Generation via CLI

```bash
# Reindex URL rewrites
bin/magento indexer:reindex url_rewrite

# Clean URL rewrites cache (after bulk operations)
bin/magento cache:clean url_rewrite
```

### Regenerating All Rewrites for a Store

```php
<?php
// Use \Magento\UrlRewrite\Model\UrlPersist to regenerate
public function regenerateStoreUrlRewrites(int $storeId): void
{
    // Get all products and regenerate
    $productCollection = $this->productCollectionFactory->create()
        ->addStoreFilter($storeId);

    foreach ($productCollection as $product) {
        $this->urlPersist->saveUrlRewrite(
            $this->urlRewriteFactory->create()->setData([
                'entity_type' => 'product',
                'entity_id' => $product->getId(),
                'request_path' => $product->getUrlKey() . '.html',
                'target_path' => 'catalog/product/view/id/' . $product->getId(),
                'store_id' => $storeId,
            ])
        );
    }
}
```

## 9. Request Path vs Target Path

### The Difference

| Field | Meaning | Example |
|-------|---------|---------|
| `request_path` | What user types or clicks | `women/jackets.html` |
| `target_path` | Where Magento internally routes | `catalog/category/view/id/12` |

- Direct rewrite: `request_path` ≠ `target_path`, `redirect_code = 0`
- Permanent redirect (301): `request_path` → `target_path`, `redirect_code = 301`
- Temporary redirect (302): `request_path` → `target_path`, `redirect_code = 302`

### Redirect Codes

```php
<?php
// 0 = Direct rewrite (no redirect, Magento handles internally)
$urlRewrite->setRedirectCode(0);

// 301 = Moved Permanently (SEO redirect)
$urlRewrite->setRedirectCode(301);

// 302 = Moved Temporarily
$urlRewrite->setRedirectCode(302);
```

## 10. Common Pitfalls and Debugging

### Pitfall 1: URL Works in One Store But Not Another

```sql
-- Check if rewrite exists for specific store_id
SELECT * FROM url_rewrite
WHERE request_path = 'jackets.html'
AND store_id = 2;  -- German store

-- If no row, URL won't resolve in German store
-- Must create store-specific rewrite
```

### Pitfall 2: Redirect Loop with Permanent Redirects

```php
// If you set redirect_code = 301 from A->B and B->A:
// Creates infinite redirect loop

// Check for conflicting rewrites:
SELECT * FROM url_rewrite
WHERE request_path IN ('page-a', 'page-b')
OR target_path IN ('page-a', 'page-b');
```

### Pitfall 3: URL Rewrite Not Regenerated After Product Save

```bash
# If product URL key didn't change, rewrite might not update
# Force regenerate:
bin/magento indexer:reindex url_rewrite

# Or clear and rebuild:
bin/magento cache:flush url_rewrite
```

### Pitfall 4: getUrl Generates Wrong Store URL

```php
<?php
// WRONG: not setting scope
$url = $this->url->getUrl('catalog/product/view', ['id' => 42]);
// Uses current store from store manager

// RIGHT: explicitly set scope for multi-store operations
$url = $this->url->getUrl('catalog/product/view', [
    'id' => 42,
    '_scope' => $storeId  // Must pass store_id
]);
```

### Debugging URL Resolution

```sql
-- Find all rewrites for a product
SELECT * FROM url_rewrite
WHERE entity_type = 'product'
AND entity_id = 42;

-- Check for broken rewrites (point to non-existent targets)
SELECT ur.* FROM url_rewrite ur
LEFT JOIN catalog_category_entity cc ON ur.target_path = CONCAT('catalog/category/view/id/', cc.entity_id)
WHERE ur.entity_type = 'category'
AND cc.entity_id IS NULL;
```

## Common Mistakes to Avoid

1. ❌ Creating rewrites with conflicting `request_path` for same store → One overwrites the other
2. ❌ Forgetting `store_id` in rewrite → Rewrite applies to all stores (may cause wrong store URL)
3. ❌ Using permanent redirect (301) for temporary changes → Confuses SEO if not reversed
4. ❌ Hardcoding URLs instead of using `getUrl()` → Breaks across stores and multi-site
5. ❌ Not regenerating URL rewrites after bulk category rename → Old URLs 404
